      Churn Prediction

      A full ML app predicting customer churn for a telecom company. Three models trained on real-world imbalanced data, served via FastAPI and explored through a multi-page Streamlit UI — with pages for live predictions, model comparison, SHAP feature importance, learning curves, EDA, and model explanations.

      Random Forest Baseline
      MLP (PyTorch) Recall 0.84
      LightGBM AUC 0.85 ✓
      Reflections
      • Data was 73/27 imbalanced — accuracy alone is misleading. A model that always predicts "No Churn" scores 73% but is useless.
      • Recall became the key metric. pos_weight helped the MLP focus on the minority class and reach recall 0.84 — but at the cost of more false positives (F1 0.62).
      • LightGBM won overall: AUC-ROC 0.85, F1 0.63. Random Forest had the worst recall (0.47) despite the highest accuracy.
      • Data leakage is easy to introduce. All preprocessing must happen after the train/test split — otherwise the model peeks at test data.
      • There is no single best metric. The right one depends on what you're actually trying to solve.

      Gotland Explorer

      How it's built and what connects it

      BrowserThis portfolio
      →
      FastAPIRender
      →
      SMHI APILive weather
      FrontendVercel
      • FastAPI · Python The backbone of the project. Exposes endpoints for weather, places, ferry schedules and the day-trip planner. All data is assembled and returned as clean JSON.
      • SMHI Open API Sweden's meteorological institute provides free open weather data. The API fetches real-time forecasts for Visby, Slite and Hemse using coordinate-based point queries. Results are cached server-side for one hour to avoid overloading SMHI's servers.
      • Places data · JSON A hand-curated list of Gotland attractions stored as a JSON file on the server. Each place has a category (nature, culture, food), coordinates, duration and website. The day-trip planner uses a greedy distance algorithm to pick the best stops within your time budget.
      • Ferry schedules · Static data Approximate departure times for Destination Gotland's routes between Nynäshamn, Oskarshamn and Visby. Served as static data — the API reminds users to verify exact times on destinationgotland.se.
      • Render Hosts the Python backend. Auto-deploys on every push to GitHub.
      • Vercel Hosts the static frontend. The portfolio fetches the Gotland Explorer API directly from the browser.

      CollectorHub

      A social platform for collectors — built together with a classmate as a school project that we plan to keep developing. Users create an account, build a personal digital showroom and add items with images, categories and descriptions. The platform makes it easy to discover other collectors, send friend requests, chat and initiate trade or buy requests.

      The stack combines React + Tailwind for a fast, responsive UI with a FastAPI backend handling all CRUD operations, database logic and API communication. AI features include image moderation on upload and optional AI-based valuation of items from image + metadata.
